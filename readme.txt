"Текстовая база данных"

# Документация 
textdb приносится в другие проекты в качестве базы данных вместо MySQL
Папка /textdb/table содержит классы-таблицы, создаваемые методом MakeTable
Папка /textdb/data содержит записи классов-таблиц




Все также, новый проект, "один файл - один класс одноименный", все подключать в index.php

--------------------

Постановка задачи: Реализовать текстовую базу данных.

Cоздать папку "data" (база данных), в ней каждая папка (название латиницей) будет таблицей, внутри этих "папок-таблиц" каждая запись будет храниться в виде отдельного файла "txt" (например 1.txt 2.txt 3.txt), содержимое файла будет представлять из себя одномерный массив конвертированный функцией serialize().


1. Объявить класс "Record", c методами "create, read, update, delete" и свойством "tableFolder". Объект этого класса не будет объявляться, только наследоваться "классами-таблицами".

- Cвойство типа строка "tableFolder" будет пустое, и будет конкатинироваться в пути в последующих методах.

- Метод "create" будет принимать параметры "данные для очередной записи (массив)" и будет создавать записи учитывающие нумерацию файлов по порядку.

- Метод "read" будет принимать параметр "номер записи" по которому будет отдавать данные из соответствующего файла "папки-таблицы" обратной конвертацией содержимого при помощи unserialize(). Если оставить пустой аргумент, вернет все записи "папки-таблицы".

- Метод "update" принимает параметры "номер записи, массив ключ-значение которые необходимо исправить".

- Метод "delete" принимает параметр "номер записи" и будет удалять файл-запись.


2. Объявить класс например "product" (тоесть это уже конкретный класс работающий со своей одноименной папкой-таблицей). Он будет наследовать класс "record".
- Объявить свойство "tableFolder" в котором будет задаваться одноименное название с текущим классом и папкой-таблицей.
- Объявить свойство "structure" (определяет структуру ключей) присвоить индексный массив, значения которого будут задавать структуру записей. 
Например при вызове метода "update" прежде чем скорректировать значение, будет проверка существования ключа in_array(ключ_значение_которого_нужно_исправить, $this->structure). Та же самая проверка нужна при создании записи.
- Также создать одноименную с классом папку в папке "data".


4. В "index.php" подключить классы и создать объект класса "product". Проверить работоспособность.
